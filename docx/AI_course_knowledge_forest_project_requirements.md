# 《人工智能原理》知识森林可视化 demo 项目需求说明

> 本文档根据“标准录音 1-2026-05-15 11-33-23”的录音转写内容整理，用于后续放入项目仓库，供 Codex / Claude Code 等代码生成工具读取。本文档的目标是完整提炼老师在录音中布置的项目任务、交付物、实现边界、技术路线和初版 demo 建议。

## 1. 项目背景

老师之前已经尝试过一个课程知识展示类项目。该项目的大体方式是：不给大模型提供额外课程资料或外部数据，而是直接依赖大模型自身已有知识，对一门课程进行知识点规划和内容生成。之前的示例中，大模型围绕一门课程整理出大约 97 个知识点，并按照章节或类别进行聚类。每个知识点下可以包含核心思想、知识对比、说明内容等。

老师认为，这种“由大模型基于自身知识生成课程知识内容”的方向是可行的。尤其对于《人工智能原理》这类课程，大模型自身掌握的相关知识通常比较丰富，不一定需要额外输入教材、课件或知识库。老师明确表示，目前这个项目不需要给模型提供外部知识或额外数据。

但是，老师对原有项目的展示方式不满意。原项目可能偏静态、偏普通目录化或偏传统页面展示，缺少活泼性和形象化表达。因此，这次项目的核心不是简单复刻旧项目，而是参考旧项目的思路，做一个更适合课程知识展示的可视化 demo。

---

## 2. 项目总目标

本项目的总目标是：

**基于大模型自身知识，为《人工智能原理》课程自动生成一个“知识森林”式课程知识可视化展示 demo，并在此基础上抽象出一套可复用的 skill，使以后切换到其他课程时，也能快速生成类似的课程知识展示系统。**

该目标包含两个层次。

第一层是初版 demo：以《人工智能原理》为对象，做出一个可以展示课程知识结构和知识点内容的可视化系统。系统应具有“知识森林”或类似的形象化结构，不只是普通的文字目录。

第二层是方法抽象：把完成这个 demo 的过程总结为若干可复用 skill，包括课程知识点规划、知识聚类、知识点内容生成、展示形式设计、图示生成、动画生成、前端集成和质量检查等。以后遇到另一门课程时，可以复用这些 skill，让 Codex 或 Claude Code 快速生成新的课程知识展示系统。

## 3. 项目不是要做什么

根据录音内容，需要明确排除以下误解：

1. 不是简单照着原来的项目复制一个。
2. 不是人工手工整理一本教材。
3. 不是一开始就做复杂的知识库、数据库或后端管理系统。
4. 不是要求所有知识点都必须有动画。
5. 不是要求一定做 3D。
6. 不是要求必须前后端分离。
7. 不是依赖爬取外部网页图片或网络素材。

老师更关注的是需求规划、可视化表达、图文并茂、后续可复用性，以及如何让大模型在这个流程中承担主要生成工作。

## 4. 第一阶段任务：完成《人工智能原理》初版 demo

### 4.1 课程对象

第一版 demo 的课程对象是：

**《人工智能原理》**

也可以命名为：

- 人工智能原理知识森林
- AI Principles Knowledge Forest
- 人工智能课程知识可视化系统

### 4.2 初版 demo 的核心功能

初版 demo 至少需要具备以下功能：

1. 展示《人工智能原理》的整体知识结构。
2. 将课程知识点划分为若干章节、主题簇或知识树。
3. 用“知识森林”“知识树”“知识节点”“知识卡片”等较形象的方式呈现，而不是普通列表。
4. 支持点击某个知识点，查看该知识点的详细内容。
5. 每个知识点详情页或详情弹窗应包含结构化信息。
6. 展示形式应包含文字、图示，部分知识点可包含动画。
7. 系统应尽量保持简单、可运行、可演示。
8. 初版不强制使用后端，可以采用静态 JSON 数据 + 前端渲染。

### 4.3 知识森林的基本设想

“知识森林”可以理解为一个课程知识可视化主界面。它可以由若干棵树组成，每棵树代表一个课程主题或章节；树上的枝叶、果实或节点代表具体知识点。用户点击节点后，可以打开对应知识点的知识屏、详情卡片或弹窗。

不必严格限制为真实树的形态。也可以采用以下变体：

- 知识森林：多个主题树，每棵树代表一个知识簇。
- 知识星图：多个知识星球或星座代表不同主题。
- 知识岛屿：不同岛屿代表不同章节。
- 知识地图：主干路径 + 分支节点。
- 知识屏系统：主界面是可视化导航，点击后进入知识屏。

老师的偏好是：稍微活泼、形象、抽象一点，可以有卡通感，但不要过度复杂。

## 6. 内容生成要求

建议初版控制在 30 到 60 个知识点之间。老师之前的示例中有约 97 个知识点，但初版 demo 不一定要做这么多。初版应以展示效果和结构完整性为优先。

建议将课程划分为以下主题簇：

1. 人工智能导论与发展历史
2. 搜索与问题求解
3. 知识表示与推理

这些主题可以作为知识森林中的树、区域或分支。

### 6.2 单个知识点内容结构

- id：知识点唯一标识

- coreIdea：核心思想

- comparisons：相关对比
- formula：关键公式或符号表达，可选
- applications：典型应用
- visualSuggestion：图示建议
- animationSuggestion：动画建议，可选
- ideologicalElement：课程思政或价值引导，可选
- difficulty：难度，例如 beginner / intermediate / advanced
- prerequisites：前置知识

## 7. 展示内容要求

### 7.1 不应只有纯文字

老师明确提出，每个知识点展示时要稍微图文并茂，有图，甚至可以有动画。初版 demo 应避免变成纯 Markdown 文档或普通表格。

### 7.2 图示要求

图示可以通过以下方式实现：

1. SVG 绘制。
2. Canvas 绘制。
3. CSS 图形。
4. 前端组件组合。
5. Mermaid 图，可选。
6. 使用模型生成的简单示意图描述，再由前端绘制。

不建议初版直接爬取网络图片，因为老师提到爬图可能涉及版权问题。更稳妥的方式是自绘示意图。例如 Transformer 可画简化结构图，强化学习可画 agent-environment 交互闭环，A* 搜索可画网格路径扩展。

### 7.3 公式要求

对于适合展示公式的知识点，可以加入公式。老师认为公式对大模型来说不是问题。前端可以使用 KaTeX / MathJax，也可以先用普通文本或简化 LaTeX 字符串展示。

适合包含公式的知识点包括：

- 梯度下降
- 反向传播
- 注意力机制
- 贝叶斯公式
- SVM 间隔
- Q-learning 更新公式
- 交叉熵损失
- Softmax

### 7.4 动画要求

动画不是所有知识点都要做。初版可以选择 2 到 4 个典型知识点实现简单动画。

适合初版动画展示的知识点包括：

1. A* 搜索：展示节点从起点向终点扩展的过程。
2. 梯度下降：展示小球沿损失曲面或曲线下降的过程。
3. 注意力机制：展示 token 之间的注意力权重变化。
4. Transformer：展示输入 token 经 embedding、attention、feed-forward 的流程。
5. 强化学习：展示智能体与环境之间观察、动作、奖励的循环。
6. 神经网络前向传播：展示信号从输入层传播到输出层。

动画应服务于理解，不要为了炫技而复杂化。

## 8. 视觉风格要求

老师希望展示方式更活泼、更形象，但不要特别复杂。初版建议采用以下风格：

- 轻量卡通风或教育可视化风格。
- 知识森林主界面有明显的主题树或知识区域。
- 色彩可稍微丰富，但整体要清晰。
- 节点、树、卡片、弹窗之间有明确层次。
- 动效适度，不要影响信息阅读。
- 不要求必须 3D。
- 适合汇报演示。

可以使用以下视觉元素：

- 树、树枝、叶子、果实、种子
- 知识卡片
- 主题徽章
- 节点连线
- 学习路径
- 进度感
- 图文知识屏
- 轻量动画组件

## 9. 初版技术路线建议

### 9.1 推荐路线

初版建议采用静态前端方案：

- Vite + React + TypeScript
- Tailwind CSS 或普通 CSS
- Framer Motion 可选，用于轻量动效
- SVG / Canvas / CSS 实现图示和动画
- 本地 JSON 文件保存课程知识数据
- 不接入后端
- 不接入数据库
- 不调用外部 API

这样可以保证项目简单、可运行、易演示，也符合老师“先简单来”“先把基础做上”的要求。

### 9.2 数据结构建议

项目中可以包含：

```text
src/
  data/
    courseKnowledge.ts
  components/
    KnowledgeForest.tsx
    KnowledgeTree.tsx
    KnowledgeNode.tsx
    KnowledgeDetailPanel.tsx
    VisualDiagram.tsx
    AnimationShowcase.tsx
  animations/
    AttentionAnimation.tsx
    GradientDescentAnimation.tsx
    SearchAnimation.tsx
  App.tsx
  main.tsx
```

如果不使用 React，也可以用原生 HTML/CSS/JavaScript，但建议使用 React 方便组件化。

### 9.3 页面结构建议

初版页面可以分为三块：

1. 顶部说明区：展示项目标题、课程名称、设计目标。
2. 左侧或中心知识森林区：展示不同知识簇和知识点节点。
3. 右侧或弹窗详情区：点击知识点后显示图文内容、公式和动画。

也可以采用全屏知识森林 + 右侧抽屉详情面板。

## 10. skill 抽象要求

老师明确提出，demo 之后要抽象出几个 skill，使以后面对其他课程时，可以快速生成类似系统。这是本项目的后续价值所在。

建议抽象以下 skill：

### 10.1 Course Planning Skill

输入：课程名称。

输出：课程章节、主题簇、知识点列表、知识点层级结构。

功能：让大模型先规划一门课程有哪些核心内容，而不是直接写页面。

### 10.2 Knowledge Cluster Skill

输入：知识点列表。

输出：知识簇、章节结构、知识森林结构、节点之间的关系。

功能：决定哪些知识点属于同一棵树、同一分支或同一区域。

### 10.3 Knowledge Content Skill

输入：知识点名称和所属课程。

输出：核心思想、基本原理、对比、公式、应用、前置知识、思政元素等。

功能：生成知识点详情内容。

### 10.4 Visual Design Skill

输入：知识点内容。

输出：适合该知识点的图示类型、页面布局、图文结构。

功能：决定该知识点如何“看得见”。

### 10.5 Diagram Generation Skill

输入：知识点和图示要求。

输出：SVG / Canvas / Mermaid / React 组件形式的图示。

功能：避免只用文字，生成自制图示。

### 10.6 Animation Planning Skill

输入：知识点内容。

输出：是否适合动画、动画目标、动画步骤、动画实现方式。

功能：判断哪些知识点适合做动画，以及动画应该展示什么。

### 10.7 Demo Integration Skill

输入：课程结构、知识点内容、图示、动画组件。

输出：可运行的前端 demo。

功能：把所有生成内容整合为一个可演示项目。

### 10.8 Quality Review Skill

输入：完整项目或生成内容。

输出：错误检查、重复检查、内容过长检查、展示一致性检查、功能可运行性检查。

功能：保证结果可汇报、可维护。

## 14. 风险与注意事项

### 14.1 内容准确性风险

由于初版主要依靠大模型自身知识生成，可能存在不准确、过泛或不够贴合课程的问题。初版汇报时可以说明：当前目标是验证生成流程和展示范式，后续可以接入课程大纲或教材进行校正。

### 14.2 图像版权风险

不要直接爬取网络图片作为项目素材。优先使用自绘 SVG、Canvas、CSS 图形或模型生成的可复用图示结构。

### 14.3 过度复杂风险

不要一开始做复杂 3D、复杂后端、复杂编辑器或完整课程平台。老师强调先简单来，先把基础做上。

### 14.4 动画失焦风险

动画应解释知识点机制，而不是为了炫技。每个动画都应能回答“它帮助学生理解了什么”。

**以《人工智能原理》为样例课程，利用大模型自身知识自动规划课程知识点、生成知识点内容，并通过知识森林、图文知识屏和少量动画实现一个可演示的课程知识可视化 demo；在此基础上抽象出一组可复用 skill，使未来可以快速迁移到其他课程。**
