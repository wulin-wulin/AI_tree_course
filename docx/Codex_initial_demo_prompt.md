# 给 Codex 的初版 demo 开发提示词

下面这段可以直接作为 Codex 的任务提示词使用。建议把本文件、录音原稿、需求说明文档一起放到项目仓库的 `docs/` 目录下，让 Codex 先阅读这些文件，再开始实现。

---

## Codex Prompt

你正在开发一个课程知识可视化 demo。请先阅读项目中的以下文档：

1. `docs/project_requirements_from_recording.md` 或类似命名的需求说明文档。
2. `docs/录音原稿.txt` 或类似命名的录音转写文件。
3. 如果项目中已有 README、设计草稿或旧代码，也请一并阅读。

你的任务是根据老师在录音中布置的要求，完成《人工智能原理》课程的“知识森林”初版 demo。请注意：这个项目不是传统课程网站，也不是简单的 Markdown 知识列表，而是一个更形象、更活泼、图文并茂的课程知识展示系统。

### 一、项目目标

请实现一个可运行的前端 demo，主题为：

《人工智能原理》知识森林可视化系统

系统需要基于大模型自身知识，内置一份《人工智能原理》的课程知识结构和知识点内容。初版不需要调用外部 API，不需要接入后端，不需要数据库，不需要爬取网络图片。

核心目标是：

1. 用“知识森林”或类似形象化方式展示《人工智能原理》的知识结构。
2. 将课程知识点划分为若干主题簇。
3. 点击知识点后，可以看到图文并茂的详情内容。
4. 部分知识点应包含公式、图示或简单动画。
5. 项目结构要便于后续抽象成 skill，并迁移到其他课程。

### 二、技术要求

如果当前仓库为空或没有明确技术栈，请创建一个 Vite + React + TypeScript 项目。

推荐使用：

- React
- TypeScript
- Vite
- CSS / Tailwind CSS（二选一，优先保证简单可运行）
- SVG / CSS / React 组件实现图示
- 可选使用 Framer Motion，但不要为了动画引入过度复杂依赖

不要接入后端。不要接入数据库。不要调用外部模型 API。不要爬取网络图片。

请保证以下命令可以运行：

```bash
npm install
npm run dev
npm run build
```

如果你修改了 package.json，请确保依赖合理、命令可用。

### 三、页面功能要求

请实现一个单页 demo，至少包含以下区域：

1. 顶部标题区
   - 显示“人工智能原理知识森林”。
   - 简要说明：由大模型规划课程知识点，通过知识森林、图文知识屏和动画展示课程结构。

2. 知识森林主视图区
   - 展示若干知识簇。
   - 每个知识簇可以表现为一棵树、一个知识区域、一个主题岛屿或一个节点群。
   - 每个知识点显示为叶子、果实、节点或卡片。
   - 点击知识点后，应打开详情面板。

3. 知识点详情区
   - 显示知识点标题。
   - 显示所属知识簇。
   - 显示一句话概括。
   - 显示核心思想。
   - 显示基本原理。
   - 显示关键术语。
   - 显示应用场景。
   - 如有公式，显示公式。
   - 如有图示建议，显示自绘图示或结构化图形。
   - 如有动画，显示简单动画组件或动态示意。

4. skill 说明区
   - 在页面底部或 README 中说明后续如何抽象成 skill。
   - 至少列出 Course Planning Skill、Knowledge Content Skill、Visual Design Skill、Animation Planning Skill、Demo Integration Skill。

### 四、课程知识结构要求

请在本地代码中创建课程数据文件，例如：

`src/data/courseKnowledge.ts`

数据至少包含 6 个知识簇、30 个知识点。建议包含以下知识簇：

1. 人工智能导论与发展历史
2. 搜索与问题求解
3. 知识表示与推理
4. 机器学习基础
5. 神经网络与深度学习
6. 自然语言处理与计算机视觉
7. 强化学习与智能体
8. 生成式 AI、AI 安全与伦理

至少包含以下典型知识点中的一部分：

- 人工智能定义
- 图灵测试
- 符号主义
- 连接主义
- 状态空间搜索
- 广度优先搜索
- 深度优先搜索
- A* 搜索
- 启发式函数
- 博弈搜索
- 命题逻辑
- 一阶谓词逻辑
- 知识图谱
- 监督学习
- 无监督学习
- 强化学习
- 决策树
- 贝叶斯分类
- 神经元模型
- 多层感知机
- 反向传播
- 梯度下降
- 卷积神经网络
- 循环神经网络
- 注意力机制
- Transformer
- 词向量
- 预训练模型
- 大语言模型
- 图像分类
- 目标检测
- 马尔可夫决策过程
- Q-learning
- 扩散模型
- AI 伦理
- 模型偏见
- 可解释性
- 可信 AI

每个知识点建议使用如下数据结构：

```ts
export type KnowledgePoint = {
  id: string;
  title: string;
  clusterId: string;
  shortSummary: string;
  coreIdea: string;
  principles: string[];
  keyTerms: string[];
  comparisons?: string[];
  formula?: string;
  applications: string[];
  visualType?: 'diagram' | 'flow' | 'network' | 'timeline' | 'formula' | 'animation';
  animationType?: 'attention' | 'gradient' | 'search' | 'agentLoop' | 'none';
  ideologicalElement?: string;
  difficulty: '基础' | '中等' | '进阶';
  prerequisites?: string[];
};
```

可以根据实际实现调整字段，但不要把知识内容写成无结构的长字符串。

### 五、图文和动画要求

请至少实现 3 种自绘图示组件，例如：

1. Transformer / 注意力机制的简化结构图。
2. 强化学习 agent-environment 交互闭环图。
3. 搜索算法节点扩展图。
4. 神经网络层级结构图。
5. 梯度下降曲线示意图。

请至少实现 1 到 2 个简单动画组件，例如：

1. 注意力机制中 token 之间的权重流动。
2. 梯度下降小球沿曲线下降。
3. A* 搜索在网格中扩展节点。
4. 强化学习中 agent 与 environment 的循环交互。

动画只需要简洁直观，不要过度复杂。动画的目标是辅助理解，而不是炫技。

### 六、视觉风格要求

整体风格应符合老师要求：

- 稍微活泼。
- 形象化。
- 可带轻量卡通感。
- 不要过度复杂。
- 不强制 3D。
- 适合课堂或项目汇报演示。
- 图文并茂，避免纯文字页面。

请优先保证布局清晰、交互明确、视觉统一。

### 七、实现步骤要求

请按以下步骤执行：

1. 先检查当前仓库结构。
2. 如果已有前端项目，在现有项目上实现；如果没有，则创建 Vite + React + TypeScript 项目。
3. 创建课程知识数据文件。
4. 创建知识森林主界面组件。
5. 创建知识点详情面板组件。
6. 创建图示组件。
7. 创建 1 到 2 个动画组件。
8. 创建或更新 README，说明项目目标、运行方式、当前实现内容和后续 skill 抽象方向。
9. 确保 `npm run build` 可以通过。
10. 最后输出你完成了哪些文件、如何运行、还有哪些后续可扩展点。

### 八、质量要求

请避免以下问题：

1. 不要只生成普通列表页面。
2. 不要只有纯文字内容。
3. 不要依赖外部图片链接。
4. 不要把所有逻辑堆在 App.tsx 中。
5. 不要引入不必要的复杂后端。
6. 不要让页面只能静态观看，至少要有点击知识点查看详情的交互。
7. 不要把动画做得过重，优先保证可运行和可理解。
8. 不要忽略 README。

### 九、初版完成标准

完成后，项目应达到以下标准：

1. 本地可以启动。
2. 页面能展示《人工智能原理》知识森林。
3. 至少有 6 个知识簇和 30 个知识点。
4. 点击知识点能查看结构化详情。
5. 至少有 3 种图示。
6. 至少有 1 个动画示例。
7. README 说明清楚运行方式和项目设计思路。
8. 项目中能看出后续迁移到其他课程的 skill 化思路。

请直接开始实现，不要只给方案。
