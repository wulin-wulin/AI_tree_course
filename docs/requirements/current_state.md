## 当前项目状态

当前项目是一个《人工智能原理》课程知识可视化 demo，采用 Vite + React + TypeScript 实现，为纯前端单页应用。项目不接入后端、数据库或外部图片资源，课程内容和展示结构主要写在 `src/data/courseKnowledge.ts` 中。

当前页面整体由三个主要区域组成：

- 顶部课程工作台 Header：展示项目名称、课程展示定位、知识簇数量、知识点数量、动画示例数量和 Demo 状态。
- 主工作区左侧 Knowledge Atlas：以可视化坐标地图的形式展示 8 个知识簇和知识点节点，用户可以通过知识簇按钮、地图枢纽、地图节点和底部学习路径按钮切换当前知识点。
- 主工作区右侧知识详情面板：会跟随左侧当前选中的知识点变化，展示学习目标、前置建议、表达形式、核心思想、基本原理、关键术语、公式、自绘图示、动画、相关对比、典型应用、前置知识和课程思政内容。
- 页面底部 SkillStrip：展示后续迁移到其他课程时可能抽象出的 Course Planning、Knowledge Content、Visual Design、Animation Planning、Demo Integration、Quality Review 等 skill 方向。

当前数据规模：

- 8 个知识簇。
- 49 个结构化知识点。
- 多种自绘 SVG 图示类型。
- 4 类轻量动画示例：注意力流动、梯度下降、A* 搜索扩展、智能体闭环。

当前主要交互：

- 点击知识簇筛选/展开地图区域。
- 点击地图中的知识点节点，右侧知识详情同步更新。
- 点击底部学习路径中的知识点，右侧知识详情同步更新。
- 图示和动画作为知识解释内容展示，但用户对这些内容本身的交互较少。

当前验证状态：

- 2026-05-25 使用 `npm.cmd run build` 验证通过。
- 直接在 PowerShell 中运行 `npm run build` 会受到本机执行策略对 `npm.ps1` 的限制；这属于本机 shell 策略问题，不是项目构建失败。

## 当前已知限制

- 左侧坐标式可视化导航有新意，但视觉上较拥挤、杂乱，尚未达到成熟产品的人类审美水平。
- 右侧知识框已经能随左侧点击切换内容，作为初版可用，但内容加载和信息呈现仍然比较生硬。
- 页面主要交互集中在“左侧选择知识点，右侧切换详情”这一条链路上，其他区域更多是静态文字编排，整体容易显得像 PPT 展示页。
- 页面底部当前展示了面向开发者的 skill 化流程设想，这不符合学习者用户视角；当前阶段不应在前端展示或强调 skill 概念。
- 当前项目仍处于 demo/玩具阶段，距离成熟的课程学习系统、知识工作台或可长期使用的教学产品还有明显差距。
