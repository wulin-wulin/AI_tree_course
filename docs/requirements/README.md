# Requirements Workflow

本目录用于维护项目迭代需求、执行交接和审核反馈。

## 文件职责

- `AGENT_PROTOCOL.md`：说明需求讨论窗口与执行智能体的协作协议。
- `current_state.md`：由需求讨论窗口维护，描述项目当前已经实现的真实状态。
- `backlog.md`：暂存尚未进入轮次的想法、问题和后续可能需求。
- `rounds/`：按轮次归档需求文档，每一轮一个文件。
- `../next_upgrade_prompt.md`：交给执行智能体的当前执行入口。

## 轮次约定

- 轮次编号使用 `R001`、`R002`、`R003`。
- 需求进入执行前，轮次状态应标记为 `Ready`。
- 执行智能体只执行 `next_upgrade_prompt.md` 指向的当前轮次。
- 执行完成后，执行智能体必须填写或追加执行反馈。
- `current_state.md` 不由执行智能体直接改写；审核确认后由需求讨论窗口维护。

## 状态约定

- `Draft`：讨论中，不能执行。
- `Ready`：已确认，可以交给执行智能体。
- `Implemented`：执行智能体已完成实现并提交反馈。
- `Reviewed`：需求讨论窗口已审核。
- `Accepted`：审核通过，当前状态文档已更新。
