# R010 - 知识森林地图重构为「沿途生长的徒步长卷」

状态：Draft

创建日期：2026-06-01

## 背景

R005–R008 把章节地图迭代成「8 棵章节树 + 蛇形小径 + 点击弹章节预览方框」的俯览扁平地图。需求讨论窗口在本轮开场对现状做了实测（桌面 1280×820 / 移动 390 宽截图）并确认了几个核心问题：

- **版面两头不讨好**：固定 1000×700 比例等比缩放，桌面首屏放不下整幅地图（第 7、8 章与「林深处」落在折叠线外），移动端地图被压成 ~259px 高的小条、树与下一行标题竖向打架，且地图下方留约 340px 空白，整体偏 demo。
- **构图右下偏空**、背景装饰树规整呆板像占位符、整幅偏「干净示意图」缺森林氛围。
- 知识点（小节）只藏在点击后的预览方框里，地图本身信息量低。

用户在本轮选择「森林地图继续打磨」方向并明确「允许调整版面」，随后给出更进一步的创意诉求（见下）。本轮据此对章节地图（Level 1，路由 `/ai`）做一次结构性重构。

## 用户真实反馈

> 原话保留：

- 「我想要让路更长一些，然后将每章的小节也放在地图上，章节之间用不同颜色的背景和不同的树进行表示。」
- 「方式2（纵向长卷向下滚动），然后需要分布均匀，不能太密集也不能太稀疏。」
- 「不要太窄，一行不能仅放两个小节，需要有蜿蜒曲折的感觉。」
- 「路需要更加圆滑一些，你需要选择一下针对这个的图形学算法。」

本轮设计经「需求讨论窗口」用浏览器可视化草图（superpowers brainstorming 可视化陪同）逐项确认：地图整体结构、滚动方向、小节节点样式、点击交互模型、桌面/移动响应式、曲线算法，均由用户在草图上逐一拍板。

## 本轮目标

把 Level 1 章节地图从「8 棵树 + 预览方框」重构为**一条贯穿全部 49 个小节、宽幅蜿蜒、向下滚动的徒步长卷**：路更长、小节直接呈现在地图上、章节以不同生态色区 + 不同树种区分，并顺带解决旧版的视口适配、构图留白与森林氛围问题。

## 具体需求

### 1. 结构与叙事
- 地图主体改为**一条连续长路**，按全课程连续顺序（章序 + 章内序）依次串联**全部 49 个知识点（小节）**。
- 顶部「林口」起步、底部「林深处」收尾；**整幅纵向滚动**，自上而下走完旅程。
- 复用现有数据：章节顺序取 `chapters`，章内小节取 `chapterPoints(chapter.id)`；跳转复用 `pointPath` / `findPoint`；进度复用 `readLastPoint`。不新增 / 不改课程数据结构。

### 2. 版面与响应式（核心约束）
- 路**大幅度横向蜿蜒、铺满主区宽度**；每一段「波浪」上排 **3–5 个小节**。**禁止**退化成「窄列里左右各一、每行仅 2 点」的形态。
- 小节节点**步距均匀**——疏密一致，不太密也不太稀疏（统一节奏感）。
- 桌面：路占满主内容宽度；两侧多余留白用**稀疏装饰森林 + 草地纵深**填满，不再空旷。
- 移动：同一条路波浪幅度收窄、保留同样的曲折感，**每行仍多于两个小节**；不再出现旧版「又挤又空」。
- 桌面与移动是同一条长路的不同尺度（不再是「固定比例整幅缩放」那种两头不讨好的方案）。

### 3. 章节表示：生态色区 + 不同树种
- 8 个章节 = 8 块**淡色生态色区**（沿路的柔和色块 / 区带），颜色取各章 `accent`（配 `soft` / `dark`）。
- 每章开头一棵明显更大的**地标树**，**8 章用 8 种不同树种剪影**，沿用现有扁平几何风格。建议初始映射（执行可微调，但须 8 种可清晰区分）：
  1. 01 导论与历史 — 层叠针叶松（teal）
  2. 02 搜索与问题求解 — 高瘦白杨（orange）
  3. 03 知识表示与推理 — 圆冠阔叶树（purple）
  4. 04 机器学习基础 — 宽冠橡树（blue）
  5. 05 神经网络与深度学习 — 柱形塔柏（pink）
  6. 06 自然语言处理与计算机视觉 — 细干桦树（teal-green）
  7. 07 强化学习与智能体 — 垂枝柳树（olive）
  8. 08 生成式 AI 与安全伦理 — 团簇矮灌丛（coral）
- 视觉层级：**地标树明显更大、树种各异；小节树苗更小、同章同色统一**，避免 49 棵小树与 8 棵地标树互相抢视觉。

### 4. 小节表示与交互（交互模型 1）
- 49 个小节 = 该章颜色的**小树苗**，沿路生长；**标题常驻**：
  - 桌面：标题在树苗旁、沿路左右交替排布，避免互相遮挡。
  - 移动：标题置于树苗**下方**、更紧凑（防止窄屏横向碰撞）。
- **点击直达**：
  - 点小节树苗 → 进入**该小节阅读页**（`pointPath`）。
  - 点章节地标树 → 进入**该章第一个小节**的阅读页。
- **悬停 / 键盘聚焦**小节或地标树 → 浮出**一张轻预览卡**（标题 + 一句梗概 / 引子）；**任意时刻最多一张**预览卡可见（延续 R009 「单预览卡」原则，鼠标移开 / 切换焦点即消失，不残留）。
- **移除 R008 章节预览大方框**：`ChapterPreviewDialog` 退役（小节已直接在地图上，方框冗余）。

### 5. 进度与状态（按小节粒度）
- 进度粒度由「按章」改为**按小节**：
  - 已读过的小节树苗 = **「已长成」**（饱满色）。判定口径：全程连续序列中，**位于「上次读到点」及其之前**的所有小节视作已长成。
  - 其余（位于上次读到点之后）= **「未长成」**（淡色）。
  - 「上次读到点」那棵 = 高亮标注 **「上次读到这里」**。
  - 无任何阅读记录时：全部为「未长成」，不显示「上次读到这里」。
- 进度来源沿用 localStorage（`readLastPoint`）。

### 6. 曲线算法（路要圆滑）
- 小径改用**向心 Catmull-Rom 样条（centripetal，α = 0.5）**，逐段转三次贝塞尔，**替换现有均匀（uniform）版 `buildTrail`**。
- 硬约束：曲线必须**精确穿过每个节点**（树苗须落在路上），故只能用插值型样条；向心参数化可避免均匀版在间距不均 / 急转弯处的过冲、尖点与自交。
- 参考实现（Barry-Goldman 非均匀切线；`alpha=0.5` 即向心，端点用首尾点复制）：

  ```
  对相邻四点 p0,p1,p2,p3：
    d01=|p1-p0|, d12=|p2-p1|, d23=|p3-p2|
    t0=0, t1=t0+d01^α, t2=t1+d12^α, t3=t2+d23^α      // α=0.5
    m1 = (p1-p0)/(t1-t0) - (p2-p0)/(t2-t0) + (p2-p1)/(t2-t1)
    m2 = (p2-p1)/(t2-t1) - (p3-p1)/(t3-t1) + (p3-p2)/(t3-t2)
    段 p1→p2 的贝塞尔控制点：
      b0=p1
      b1=p1 + (t2-t1)/3 · m1
      b2=p2 - (t2-t1)/3 · m2
      b3=p2
  ```
  （需对各分母做零保护；起点 p0:=p1、终点 p3:=p2。）

### 7. 氛围与背景
- 草地做出纵深、装饰背景树疏密有致，去掉「干净示意图」感，更像一片真森林；装饰树为纯视觉、`aria-hidden`、不承载章节信息、避开路径与节点。

### 8. 顶部与导航
- 保留「返回书架」入口；滚动时做成**纤细常驻顶栏**，含轻量进度指示（如「走到第 N / 49 棵」或细进度条）。

### 9. 可达性
- 保留语义化等价导航：用有序列表（`<ol>`）按全程顺序承载 49 个小节，**键盘可按顺序 Tab**、聚焦时显示该节预览卡。
- 保留可见焦点环与必要 ARIA；点击区域为真实可聚焦控件（`<button>` / `<a>`）。
- 尊重 `prefers-reduced-motion`：滚动/悬浮动效在该偏好下降级。

## 非目标

- 不改 Level 2 阅读页的内容分层与交互；不改首页书架（Level 0）。
- 不改课程数据结构与路由（HashRouter 三级结构不动）。
- 不引入新依赖、不接入后端 / 外部图片。
- 不在本轮新增自测、概念关联探索等导航以外的学习交互（留待后续轮次）。
- 不重做 R008 方框逻辑本身——而是**整体移除**它（`ChapterPreviewDialog` 退役）。

## 验收标准

> 验收方式沿用：人类用户**主观评审** + 截图自检。

- 地图为**一条连续长路**，按全课程顺序串起**全部 49 个小节**；自上而下纵向滚动可走完「林口 → 林深处」。
- 路**宽幅蜿蜒铺满宽度**，每段波浪 3–5 个小节；**没有**退化成每行仅 2 点的窄列形态；节点**步距均匀**。
- 桌面首屏不再裁切错位、两侧不空旷；移动端不再「又挤又空」，每行仍多于两个小节、标题不互相遮挡。
- 8 章各有**可区分的生态色区 + 不同树种地标**；小节树苗更小、同章同色，不抢地标视觉。
- 点小节 → 对应阅读页；点章节地标树 → 该章首点；悬停 / 聚焦浮出预览卡且**任意时刻最多一张**；旧的章节预览大方框已移除。
- 进度按小节生效：已读「已长成」、未读「未长成」、上次读到的高亮「上次读到这里」。
- 小径曲线肉眼圆滑、无明显过冲 / 打结，且每个树苗都落在路上（向心 Catmull-Rom）。
- 键盘可按顺序 Tab 过全部小节并触发预览；焦点环可见；`prefers-reduced-motion` 下动效降级。
- `npm run build`（`tsc -b && vite build`）通过。
- 按 `AGENT_PROTOCOL.md` 第 7 步**自检视觉**：Playwright 截 `/#/ai` 桌面 + 移动并滚动走查，自己核对达标后再报告完成。
