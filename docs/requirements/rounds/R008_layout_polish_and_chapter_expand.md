# R008 - 首页/地图版面比例打磨 + 章节节点点击展开知识点预览

状态：Accepted

创建日期：2026-05-28

确认日期：2026-05-28

## 背景

R007 森林地图被接受并提交为基线。继续往后还有三处人类用户反馈：
- 首页书架（Level 0）课程卡片偏小、且页面底部有一段白块和整体不协调，比例需要调整。
- 章节地图页（Level 1 `/ai`）的绿色地图区相对网页偏小，页面整体留白偏多，地图可以撑开。
- 缺少「先看本章有哪些知识点」的中间预览：希望点击章节节点（树）后弹出一个方框——中央显示章节名称、四周显示该章的知识点、知识点与方框之间用虚线连接。

R008 集中处理这三件事。

## 用户真实反馈

> 原话保留：

1. 开头界面下面的卡片做成一字排开的，卡片可以稍微大一点；页面底部有一段白块和页面不协调，调整一下；整体页面布局比例再协调一下。
2. 地图导览页可以做大一点，中间的绿色地图页直接全部撑开，就是整个网页别留太多空白。
3. 加一个功能：鼠标点击对应章节节点之后会显示出来一个方框，方框中间显示章节名称，四周显示对应的知识点，知识点和方框之间用虚线连接。

## 已确认的关键决策

- 三件事归在同一轮做（统一打磨页面比例 + 新增章节节点展开预览）；保持 R007 森林地图基线不被推翻。
- 点击章节树的行为**从「直接进入该章首个知识点」改为「弹出本章节展开预览方框」**；用户在方框中选择具体知识点再进入阅读页（路由仍是既有 `/ai/:chapterId/:pointId`）。

## 本轮目标

让整站三级页面的版面比例都更协调（首页一字排开、底部不留违和白块；地图页撑满更少空白），并在森林地图上新增一个本章节知识点的展开预览方框（点击树打开、四周列出知识点、虚线连接到中央章节名），使章节→知识点的浏览有「先看清再进入」的中间环节。

## 具体需求

### A. 首页书架（Level 0 `/`）版面打磨

- 课程卡片由当前的多列网格改为**一字排开**：4 张课程卡（人工智能原理 + 离散数学 + 编译原理 + 强化学习的数学原理）在桌面端**横向一行**展示。
- **卡片稍微大一点**：在保持扁页气质前提下，宽度/内边距/标题字号都适度上调，让单卡更有分量、好点；不要膨胀到挤压。
- **去掉底部不协调的白块**：检查并消除「页面底部留出一段与整体配色不协调的白色 / 浅色区块」的问题——通过统一页面背景、调整 `min-height`、收尾留白等方式让页面收口干净。
- **整体比例协调**：标题区、说明文字、课程一字排开三段在视觉上节奏合理，不头重脚轻、不无谓拉长；桌面端首屏即可看见全部课程卡。
- **响应式**：窄屏 / 移动端一字排开自然回流（如纵向叠放或两两排），仍然好看、不拥挤。

### B. 章节地图页（Level 1 `/ai`）撑满

- 让中间的绿色森林地图区**撑开**：在保持 R007 蛇形布局与比例的前提下，地图卡占据更大的页面幅面（更接近首屏可用空间），减少四周和底部多余的空白。
- **整页留白克制**：标题区到地图卡的间距、地图卡到页脚的间距收紧合理；页面整体看上去「地图就是页面主体」，而不是浮在一大片空白里。
- 保留 R007 已有视觉：3/3/2 蛇形折返、扁平几何树、配色、俯览底景、林口/林深处、序号、预览卡、进度点亮、`cqw` 等比缩放。
- 响应式：窄屏 / 移动端同样让地图卡占满可用宽度，留白克制、保持匀称。

### C. 章节节点点击展开本章知识点预览方框（新功能）

> 行为改变：点击章节树**不再直接跳转**到该章首个知识点；改为弹出本章预览方框，用户在方框中选择具体知识点再进入阅读。

- **触发**：在森林地图上点击或键盘激活任一棵章节树。
- **方框（弹层）**：
  - 在地图上方以弹层（popover / modal-on-map）出现；可选锚定到被点的树或居中显示，由执行按好看优先取舍。
  - 视觉风格延续 R007 扁平几何 / 现代语言：圆角矩形、克制阴影、章节 `accent` 配色作为强调；不堆装饰。
  - **方框中央**显示章节名称（标题 + 副标题，章节序号也可保留）。
  - **方框四周**显示该章节包含的所有知识点：每个知识点为一个小卡片 / 标签（知识点标题，可带难度小标），围绕中央排布。
  - **虚线连接**：每个知识点小卡片与中央章节名之间用一条**虚线**相连，形成「中心 — 四周辐射」的关系图。
  - 知识点数量按章节不同有差异（约 4–10），布局须能自动适配数量（如圆形/椭圆形/放射状自动排布），不论多少个都不互相挤压、不被裁剪。
- **交互**：
  - 点击 / 键盘激活任一**知识点小卡片** → 进入 `/ai/:chapterId/:pointId` 阅读页（沿用 R003 路由）。
  - 中央章节名可选作为「从本章第一个知识点开始读」的快捷入口（推荐，但执行可按视觉判断；若不做也允许）。
  - **关闭方式**：点击方框外背景、按 ESC、点 X 关闭按钮、或点击另一棵树（自然切换到那棵树的方框）。
  - 打开 / 关闭有平滑的小动效（淡入 + 轻度缩放；尊重 `prefers-reduced-motion`）。
- **可达性**：
  - 方框打开时焦点移入方框内首个可聚焦元素，关闭后焦点回到被点的树。
  - ESC 关闭；Tab 在方框内循环（焦点陷阱）。
  - 方框使用合适的 `role`（如 `dialog` 或 `menu` + `aria-modal`）与 `aria-label`/`aria-labelledby`。
  - 虚线为装饰、`aria-hidden`，不干扰读屏。
- **响应式**：窄屏 / 移动端可降级为更紧凑的弹层（如纵向列表 + 虚线点缀，或居中 sheet），保留「中心 + 四周 + 虚线」的视觉意图、可点击可读。
- **与既有「悬停预览卡」的关系**：现有的悬停 / 聚焦预览卡（章节梗概 + 知识点数量）可保留作为「未点击时的快速一瞥」，也可由执行判定弱化或移除（因新方框已承载更丰富的章节预览）；以视觉协调为准。
- **路由 / 数据**：不新增路由；本章知识点列表从既有 `clusters` + `knowledgePoints`（按 `clusterId`）派生，不动数据结构。

### 执行中发现的问题

- 章节树由 `<Link>`(`<a>`) 改为 `<button>` 后会带入浏览器默认控件样式（背景、边框、字体、颜色），已在 CSS 中追加 `button.forest-tree { background:transparent; border:0; color:inherit; font:inherit; cursor:pointer; }` 重置，外观与 R007 保持一致。
