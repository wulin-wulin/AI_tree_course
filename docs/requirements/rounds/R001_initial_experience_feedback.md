# R001 - 初次体验反馈与交互成熟度提升

状态：Draft

创建日期：2026-05-25

## 背景

本轮用于归档人类用户对当前初版 demo 的真实体验反馈，并为后续正式需求讨论提供材料。

当前文档仍是 `Draft`，不是可执行需求。执行智能体读取到本轮状态为 `Draft` 时，应停止实现。

## 用户真实反馈

1. 左侧坐标式可视化导航目录挺有新意，但是有些拥挤杂乱，不符合人类审美。右侧知识框会跟随左边点击到的知识点变化并加载内容，作为初版已经很好，但是还有很大的优化空间，现在很生硬。
2. 整个页面除了左侧导航地图和右侧知识框两个模块，没有任何其他有效交互。页面只是把一些词编排到了旁边，显得很生硬，像在做 PPT，这一点后面一定要想办法改掉。
3. 整个项目现在还是个玩具，完成度很差，后面还需要继续大幅改进。

## 本轮目标

待继续讨论。

初步方向是把上述真实体验反馈转化为一轮可执行的设计与交互升级需求。目前已确认第一个硬需求：前端必须从学习者视角出发，移除所有面向用户展示的 skill 化内容。

## 具体需求

- 已确认：项目当前阶段是面向学习者的课程学习产品，不是面向开发者的生成流程展示。因此前端页面中不能出现 `skill`、`可复用 skill 流程`、`可迁移 skill 流程`、`Course Planning Skill`、`Knowledge Content Skill` 等开发者内部概念。
- 已确认：删除页面底部的可复用 skill 流程展示区域。该区域属于开发者后续可能关注的工程抽象，不是学习者用户关心的内容。
- 已确认：删除或清理前端 Header、底部模块、用户可见文案中关于 skill 化、迁移到其他课程生成流程的表达。
- 已确认：当前阶段不需要考虑后期形成 skill 的事情。相关内容不进入产品前端，也不作为本轮产品目标的一部分。后续如果需要重新考虑，由人类用户另行提出。
- 已确认：如果项目文档或 README 中把 skill 化作为当前产品后续目标，也需要降级或移除，避免执行智能体继续围绕 skill 方向扩展当前产品。
- 待讨论：是否优先改 Knowledge Atlas 的拥挤和杂乱问题。
- 待讨论：是否优先改右侧知识详情面板的内容呈现方式。
- 待讨论：是否新增更有学习意义的交互，而不只是增加装饰性动效。
- 待讨论：如何定义“从玩具感走向成熟课程学习产品”的第一阶段验收标准。
