# R003 - 三级导航范式重构（书架 → 章节路径 → 知识点阅读）

状态：Accepted

创建日期：2026-05-28

确认日期：2026-05-28

## 背景

R002 把单个知识点重构为聚焦阅读页，验收通过。但项目整体仍停留在「一进页面就是工作台 Header + 坐标式 Atlas + 详情面板」的单层结构上：没有课程层、章节导览拥挤杂乱、缺少清晰的「从课程到章节到知识点」的递进。

本轮按人类用户提出的范式，整体重构信息架构（IA）为**三级递进导航**，让产品从「一张拥挤的 demo 页」变成「有层次、可逐级深入的课程学习应用」。本轮是一次较大幅度的结构性改动（区别于前两轮的小范围迭代）。

本文件当前为 `Draft`，仅用于沉淀讨论。执行智能体读到 `Draft` 时应停止实现。

## 用户原始想法

> 原话保留，作为本轮意图依据：

网页点进去，直接是一个整体页面，标题写「知识森林学习笔记」，下面有几个课程按钮：「离散数学」「编译原理」「人工智能原理」「强化学习的数学原理」等。当前只有「人工智能原理」有内容，其他按钮只占位、不需要内容。点击「人工智能原理」进入章节导览页面，展示这门课的章节，章节做成一张顺序的序列地图。点中一个章节，进入知识点信息展示界面，刚才的导览界面变成一个章节列表悬浮在一边、且不遮挡知识点内容，界面展示该知识点对应的教学内容。

## 设计判断（讨论结论）

三级递进导航：

```
① 首页 / 扉页              ② 章节导览                ③ 知识点阅读
「知识森林学习笔记」         人工智能原理的章节          教学内容展示
课程按钮列表          →     线性学习路径（顺序序列）  →  复用 R002 阅读页
（仅 AI 原理可进入）                                   章节内知识点列表悬浮在侧、不遮挡正文
```

这套结构同时解决多个历史问题：
- 首页课程列表**替掉**带 stat 计数器的「工作台 Header」（AI 味来源之一）。
- 章节线性路径**替掉**拥挤杂乱的坐标式 Knowledge Atlas（拥挤问题的根）。
- 第三级**复用** R002 已完成的知识点聚焦阅读页；悬浮章节/知识点列表是 R002「专注阅读 / 目录退场」思路的进化。

数据映射：现有 `clusters`（8 个知识簇）即「章节」，`knowledgePoints`（49 个）即「知识点」，沿用现有数据结构，不改 `courseKnowledge.ts`。

「知识森林」叙事：整体是「学习笔记」；每门课是一片森林；章节是林间路径上的节点；知识点是落到具体一处翻开的笔记。叙事仅作轻量点缀（首页、转场），不在功能区堆砌森林装饰，避免变成新的装饰性 AI 味。

## 已确认的关键决策

- **Q1 章节 → 知识点**：点击章节**直接进入该章第一个知识点的阅读页**；悬浮列表展示**本章节的知识点**，点哪个跳哪个（复用 R002 的上一点/下一点序列）。
- **Q2 章节导览形式**：**线性学习路径**——8 个章节连成一条有起点/终点的路径，带序号与先后顺序，呼应「森林小径」。
- **Q3 路由**：**引入真实路由**，URL 反映 课程/章节/知识点，浏览器前进后退可用、刷新不丢位置、链接可分享。形式确认采用 `react-router-dom` 的 **HashRouter**。
- **章末推进**：到本章最后一个知识点时，「下一点」**自动进入下一章的第一个知识点**（全课程连续学完），不在章末停住。
- **学习目标区**：维持现状，**不**新增独立的「学习目标」区（R002 已移除的模板学习目标文案不再恢复）。

## 本轮目标

把单层页面重构为三级递进导航应用，三级各自做到位、整体串联自然。

### 一、首页 / 扉页（Level 0）

- 标题「知识森林学习笔记」，安静的扉页/书架气质，留白充足，不做 dashboard、不放 stat 计数器。
- 课程按钮/卡片列表，至少包含：人工智能原理（可进入）、离散数学、编译原理、强化学习的数学原理（占位）。
- 占位课程呈现明确的**未开放 / 敬请期待禁用态**，不可点击进入，不需要任何内容；视觉上不能显得「坏了」。
- 点击「人工智能原理」进入其章节导览。

### 二、章节导览（Level 1）

- 展示人工智能原理的 8 个章节，做成**线性学习路径**：有起点终点、带序号、能看出先后顺序。
- 每个章节节点展示章节标题（沿用 cluster 的 title/subtitle），点击进入该章第一个知识点的阅读页。
- 能看出当前所在/上次所在位置（轻量即可，不要求完整学习进度系统）。
- 提供返回首页的入口。

### 三、知识点阅读（Level 2）

- 复用 R002 的知识点聚焦阅读页（引子 → 核心思想 → 图/动画 → 正文 → 参考折叠 → 延伸）作为正文区。
- 进入时默认落在所点章节的**第一个知识点**。
- **悬浮章节列表**：展示当前章节的知识点列表，悬浮在侧、**不得遮挡正文阅读区**；点击其中知识点切换正文；可收起/展开。
- 列表中提供**切换章节**与**返回章节导览 / 首页**的入口（具体形态执行时按设计风格定，需保证可达）。
- 上一点/下一点推进沿用 R002 的全课程连续序列：到本章末「下一点」自动进入下一章首个知识点（届时悬浮列表也切到新章节）。

### 四、路由

- 引入路由，建议采用 `react-router-dom` 的 **HashRouter**（纯静态部署，深层链接刷新无需服务器重写）。
- URL 结构建议：`/`（首页）→ `/ai`（人工智能原理章节导览）→ `/ai/:chapterId/:pointId`（知识点阅读页）。
- 点击章节进入 `/ai/:chapterId` 时自动定位到该章首个知识点（重定向到带 pointId 的地址）。
- 刷新任意层级地址都能恢复到对应界面；浏览器前进/后退在三级间正确工作。
- 非法/不存在的地址有合理兜底（回首页或对应上一级）。

### 五、转场与串联

- 三级之间应有自然转场，弱化硬切（书架 →「走进」森林 → 章节路径 →「翻开」知识点）。
- 转场需尊重 `prefers-reduced-motion`。

## 非目标

- 不为占位课程（离散数学、编译原理、强化学习的数学原理）提供任何真实内容。
- 不改动 `src/data/courseKnowledge.ts` 的数据结构与课程内容（仅改变组织与呈现）。
- 不做完整的学习进度/完成度追踪系统（章节导览的「位置感」做轻量即可）。
- 不在本轮重做知识点阅读页的内部排版（R002 成果直接复用，仅做必要适配）。
- 不新增自测、概念关联探索等新的学习交互形态。
- 不新增独立的「学习目标」区（维持 R002 现状）。
- 不引入后端、数据库或外部图片资源；除路由库外不引入其他重型依赖。

### 已完成需求

- **首页书架（Level 0）**：安静扉页气质、留白充足、无 stat 计数器；课程卡片含人工智能原理（可进入，「走进森林」）与离散数学 / 编译原理 / 强化学习的数学原理（虚线边框 + 锁图标 + 「敬请期待」明确禁用态，不可点击、视觉不显得坏掉）。

- **知识点阅读（Level 2）**：复用 R002 阅读分层（引子 → 核心思想 → 图/动画 → 正文 → 参考折叠 → 延伸）；进入章节默认落在首个知识点。

- **上一点/下一点连续推进**：沿用 R002 全课程连续序列；到本章末尾「下一点」自动进入下一章首个知识点，路由随之切换、悬浮列表同步切到新章节。
- **路由**：HashRouter，`/`→`/ai`→`/ai/:chapterId/:pointId` 三级；`/ai/:chapterId` 自动重定向到该章首个知识点；非法地址兜底（知识点不存在回 `/ai`、章节与知识点不匹配重定向到正确地址、未知路径回 `/`）；刷新任意层级可恢复、浏览器前进/后退在三级间正确工作。
