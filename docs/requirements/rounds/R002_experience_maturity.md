# R002 - 交互与体验成熟度提升

## 背景

R001 已完成前端 skill 化内容的移除，并归档了人类用户对初版 demo 的真实体验反馈。R002 承接这些尚未解决的体验问题，目标是把项目从“玩具 / PPT 感”推向更成熟的课程学习产品。

本文件当前为 `Draft`，仅用于沉淀讨论，尚未确定可执行需求。执行智能体读到本轮状态为 `Draft` 时，应停止实现。

## 用户真实反馈

> 沿用 R001 归档的初版反馈，作为本轮讨论起点：

1. 左侧坐标式知识地图有新意，但拥挤杂乱、不符合人类审美。
2. 右侧知识详情面板能跟随左侧切换内容，初版可用，但内容呈现生硬，优化空间大。
3. 页面除左侧导航和右侧详情外，几乎没有其他有效交互；内容更多是把词编排在旁边，像在做 PPT。
4. 整体仍是玩具，完成度差，需要继续大幅改进。

## 设计判断（讨论结论）

数据本质是一棵层级树（课程 → 章节 → 知识点），但真正价值在叶子（每个知识点的讲解 + 图 + 动画）。当前“左导航 + 右详情”双栏同屏，让导航与阅读互相抢注意力，是“拥挤”和“像 PPT”的共同根因。

当前“AI 味”主要来自：顶部 stat 计数器（簇/点/动画数量罗列）、“工作台/workspace”框架、详情面板把约 15 个字段全量平铺成等权小标题、装饰性而非信息性的配色。核心病灶是把“信息密度”当成价值。

**已确认的成熟交互模型方向：阅读优先 · 聚焦模式。** 读某个知识点时，界面就是一篇有节奏、有层次的“页”，章节树退成安静的索引；知识点内部用渐进展开取代字段平铺，图/动画作为版面中心。概览看结构、聚焦读内容两种模式分离（延续既有“概览/聚焦双模式”方向）。

## 本轮目标

本轮交付节奏：**聚焦单点做深**。范围锁定**单个知识点的聚焦阅读体验**，做透、可验收。

把知识点页从“字段平铺面板”重构为有阅读层次的“页”，遵循 **先给意义 → 给画面 → 讲清楚 → 留参考** 的主线：

| 层 | 内容 | 处理 |
|----|------|------|
| 引子 | 标题 + 难度 + 一句话定位（shortSummary） | 安静的元信息行，不用 stat 色块堆叠 |
| 核心思想 | coreIdea | 大字、留白，像文章导语，先说清“为什么学它” |
| 画面 | 图示 / 动画 | 紧跟核心思想、作为版面中心，而非沉到底部附属 |
| 正文 | principles、formula | 有节奏地展开；公式带解释，嵌在原理叙述中，不孤立成块 |
| 参考（按需） | keyTerms、comparisons、applications、prerequisites | 降权：折叠或安静的次级区，需要才展开，不与正文抢戏 |
| 延伸 | ideologicalElement | 低调收尾 |

核心原则：**不是所有字段都平等**，把“理解一个概念”的主线立起来，其余退成支撑。

## 具体需求

- 重构知识点页（当前 `KnowledgeDetailPanel`）的信息层次，按上表组织内容，取代等权小标题平铺。
- 核心思想（coreIdea）作为页面导语层，强化排版层次与留白，建立明确的视觉重心。
- 图示 / 动画提升为版面中心元素，紧随核心思想呈现，而非排在所有文字之后。
- 次级字段（关键术语、对比、典型应用、前置知识）做**渐进展开 / 降权处理**，默认不与正文等权铺开。
- 提供**上一知识点 / 下一知识点**的连续推进，并给出当前位置感（在所属章节/课程中的位置）。
- 阅读知识点时，章节导航可退场 / 收起，让阅读区占据主视野；需要看结构时再回到概览。
- 切换知识点时的内容呈现应自然连贯，弱化当前生硬的整块替换感。

## 非目标

- 不在本轮重做概览知识地图（Knowledge Atlas）的布局与视觉，留待后续轮次。
- 不在本轮新增自测、概念关联探索等新的学习交互形态（仅做阅读体验本身）。
- 不在本轮重做顶部 Header / 移除 stat 计数器等知识点页以外的“去 AI 味”改造（记录在案，后续轮次处理）。
- 不改动 `src/data/courseKnowledge.ts` 的数据结构与课程内容（仅改变呈现方式）；如确需调整字段，先在执行反馈中提出。
- 不引入后端、外部图片或新的重型依赖。

### 已完成需求

- **信息层次重构**：知识点页按「引子 → 核心思想 → 画面 → 正文 → 参考 → 延伸」组织，取代等权小标题平铺。
