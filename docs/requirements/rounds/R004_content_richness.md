## 背景

R003 完成三级导航重构后，结构清晰、简洁，但人类用户反馈整体偏空、显得「没什么内容」。其中 Level 1 章节导览目前是一条朴素的纵向线性清单，信息感最弱、最显空。

本轮把 **Level 1 章节导览的展示形式整体换掉**：从「线性清单」改为**一张知识森林地图**——地图上一条弯曲的小径把若干棵树串起来，**每棵树代表一个章节**。地图天然比清单更有内容感与场景感，既呼应「知识森林」主题，又直接缓解「太空」的观感。

本文件当前为 `Draft`，仅用于沉淀讨论。执行智能体读到 `Draft` 时应停止实现。

## 用户真实反馈与想法

> 原话保留：

- 「（三级结构）确实很简洁，但是也太空了，就好像没什么内容。」
- 「把章节展示目录的展示形式换掉，换成一张地图，地图上弯曲的线性路径将知识点串起来，每个知识点就是一棵树。」

## 已确认的关键决策

- **粒度**：地图上**一棵树 = 一个章节**（共 8 棵）。保持现有三级结构不变：点击一棵树 → 进入该章节第一个知识点的阅读页（沿用 R003 行为与路由）。
- **形式**：弯曲的线性小径把 8 棵树按章节顺序串起来，有起点、有终点，先后一目了然（区别于已废弃的散点式坐标 Atlas）。
- **范围**：本轮只重做 Level 1（`/ai` 章节导览页）。首页书架（Level 0）与知识点阅读页（Level 2）不在本轮重做范围内。

## 本轮目标

把 `/ai` 章节导览页从线性清单重构为一张**有序、不拥挤、有内容感的知识森林地图**：8 棵树（章节）沿一条弯曲小径排列，承载更多信息与场景感，同时严格避免回到旧 Atlas 的拥挤杂乱。

## 具体需求

- 重做章节导览页（`ChapterMapPage`）为森林地图：弯曲小径串起 8 棵树，每棵树对应一个章节，带序号 01–08、能清楚看出先后顺序。
- 每棵树是**可点击/可聚焦的导航入口**，点击进入该章节第一个知识点的阅读页（路由行为沿用 R003）。
- 地图需呈现章节的关键信息（标题、副标题，以及下列优化点中确认采用的内容），让该页「有内容」而非只是图形装饰。
- 提供返回首页书架的入口。
- 保留并融入 R003 已有的「上次读到这里」位置记忆（localStorage），在地图上标注当前/上次所在章节。

## 设计优化点（执行智能体可参考采用，最终视觉由执行实现 + 主观评审）

1. **树的视觉承载信息**：用树的大小 / 枝叶繁茂度等映射该章节的知识点数量（或难度构成），让视觉是「有意义的」而非纯装饰，呼应「不堆 AI 味装饰」的原则。
2. **小径承载旅程与进度**：小径有「林口（起点）→ 林深处（终点）」的方向感；已访问 / 上次读到的章节的树呈现「点亮 / 长成」态，当前位置高亮，给出学习旅程感。
3. **悬停 / 聚焦预览**：指向或聚焦一棵树时，浮出小卡片显示章节标题、副标题、一句梗概、知识点数量等，提升信息密度但不长期占屏（直接缓解「太空」）。
4. **有机的手绘地图质感**：弯曲、略不规则的小径与形态各异的树，营造「探险 / 绘本地图」气质，而非流程图式排列；这是对抗「空、生硬」的关键。
5. **章节配色入景**：复用现有 cluster 的 accent 配色作为对应树 / 林区色调，让颜色承载「这是哪一章」的信息。
6. **响应式与不拥挤**：桌面端从容布局，移动端地图优雅缩放或纵向蜿蜒滚动；务必避免重蹈旧坐标 Atlas 的拥挤覆辙，始终保持有序、可呼吸。
7. **可达性兜底**：每棵树是语义化、可键盘聚焦与操作的按钮；为纯视觉地图提供文字 / 列表兜底或等价导航，不把读屏 / 键盘用户排除在外。

## 非目标

- 不改变三级导航结构与路由（树仍是「进入章节首个知识点」的入口）。
- 不重做首页书架（Level 0）与知识点阅读页（Level 2）。
- 不改动 `src/data/courseKnowledge.ts` 的数据结构与课程内容（地图所需信息从现有 cluster / knowledgePoints 派生）。
- 不把地图做成又一个拥挤散点图；不引入装饰性但无信息的堆砌。
- 不引入后端、外部图片资源或新的重型依赖（图形以 SVG / CSS 自绘为主）。
