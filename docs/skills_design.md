# 知识森林 demo 的 skill 抽象设计

本 demo 的后续目标不是只服务《人工智能原理》，而是沉淀一组可复用 skill，让其他课程也能快速生成类似的知识森林系统。

## 1. Course Planning Skill

输入课程名称、课程层级或教学对象，输出主题簇、章节结构、知识点清单和学习路径建议。

适合产物：

- `clusters`
- `knowledgePoints` 初始列表
- 知识点难度与前置关系

## 2. Knowledge Content Skill

输入课程名称和知识点名称，输出结构化知识内容。

建议字段：

- 一句话概括
- 核心思想
- 基本原理
- 关键词
- 相关对比
- 公式
- 应用场景
- 课程思政或价值引导

## 3. Knowledge Cluster Skill

输入知识点列表，输出更适合可视化呈现的聚类结构。

该 skill 需要判断：

- 哪些点属于同一棵树
- 哪些点适合作为主干、分支或叶子
- 哪些点存在前置依赖或横向关联

## 4. Visual Design Skill

输入知识点内容，输出图示类型和知识屏布局。

可选图示类型：

- 搜索网格
- 神经网络层级图
- Transformer 流程图
- 强化学习闭环
- 知识图谱
- 视觉任务示意图
- 可信 AI 维度图

## 5. Diagram Generation Skill

输入图示类型和知识点信息，输出 SVG、Canvas 或 React 组件。

重点约束：

- 不爬取外部图片
- 图示服务于理解
- 统一视觉语言
- 保持组件可复用

## 6. Animation Planning Skill

输入知识点内容，判断是否值得做动画，并给出动画目标、状态变化和实现方式。

适合动画的机制：

- A* 搜索节点扩展
- 梯度下降路径
- 注意力权重流动
- 智能体与环境交互闭环
- 神经网络前向传播

## 7. Demo Integration Skill

输入课程数据、图示组件和动画组件，输出可运行前端 demo。

集成检查项：

- `npm run dev` 可启动
- `npm run build` 可通过
- 点击节点可打开详情
- 图示和动画不依赖外部服务

## 8. Quality Review Skill

输入完整项目，输出质量检查结果和修改建议。

检查重点：

- 知识点数量是否达标
- 内容是否结构化
- 是否出现纯文字堆砌
- 图示是否和知识点匹配
- 动画是否帮助理解
- README 是否说明运行方式
