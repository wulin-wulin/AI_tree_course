# 下一步优化提示词：成熟学习界面与知识地图升级

## 背景问题

当前 demo 已能运行并展示《人工智能原理》知识结构，但存在两个需要持续优化的问题：

1. 界面不应像低龄卡通演示页，而应更接近成熟课程学习系统、知识工作台或课堂汇报工具。
2. “知识森林”不应停留在普通树形装饰，而要成为真正帮助学习者理解课程结构、前置关系和重点机制的知识展示设计。

## 优化目标

请将系统升级为“AI Principles Learning Studio”：

- 顶部呈现课程工作台状态，而不是宣传型 landing hero。
- 主视图采用交互式知识地图，表达主题簇、学习路径、知识节点和当前展开主题。
- 详情区采用知识屏/讲义式结构，突出学习目标、前置建议、表达形式、核心思想、原理、图示、公式和动画。
- 视觉风格克制、专业、有教育产品质感，避免儿童化树木、过度卡通、过强装饰性渐变。
- 保留知识森林隐喻，但把“森林”理解为课程知识生态、路径和主题区域，而不是简单画几棵树。

## 具体执行要求

1. 将 Header 改为课程学习工作台头部，减少巨大标题和幼稚感。
2. 将知识森林主视图升级为 Knowledge Atlas：
   - 主题簇为地图枢纽。
   - 当前主题簇展开知识节点轨道。
   - 用路径线表示课程学习路线。
   - 底部提供当前簇学习路径快速跳转。
3. 右侧详情区增加学习提示：
   - 学习目标
   - 前置建议
   - 表达形式
4. 保持自绘 SVG 图示和动画，不使用外部图片链接。
5. 保证 `npm run build` 通过。

## 设计原则

- 专业感优先于装饰感。
- 信息层级优先于视觉热闹。
- 动画只解释机制，不做无意义炫技。
- 知识结构要能服务课堂讲解：从课程总览，到主题簇，再到知识点详情。
